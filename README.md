# Тестовое задание: React Frontend-разработчик в компанию Enjoy-pro

## Стек технологий

- TypeScript
- React 16
- Webpack + npm
- Ant Design 5 и styled-components
- React Router 6
- TanStack Query 4 + axios
- dayjs
- Git / GitHub
- Архитектура FSD (Feature-Sliced Design)

## Цель проекта

Реализовать базу пользователей с возможностью:

- авторизации
- просмотра списка пользователей
- создания и редактирования пользователей через модальные окна
- обработки ошибок от сервера
- работы с датами через dayjs

## Структура приложения

### Страница авторизации (`/login`)

- Пользователь: `admin` / `admin`
- Валидация логина и пароля
- Имитация авторизации через промис с таймаутом 2000мс
- Использование TanStack Query и axios
- Ошибки отображаются через блок или notification
- Кнопка авторизации блокируется во время загрузки
- Токен сохраняется в `localStorage`
- Защита маршрутов: авторизованный пользователь автоматически перенаправляется на `/users`

### Страница 404 (`*`)

- Использован компонент `Result` из Ant Design

### Страница пользователей (`/users`)

- Получение списка пользователей через mockapi.io
- Кнопка «Выход» — разлогинивает пользователя
- Кнопка «Создать пользователя» открывает модальное окно
- Клик по аватару или имени — открывает модальное окно редактирования
- Даты отображаются в формате `DD.MM.YYYY` с помощью dayjs

### Модальные окна создания и редактирования пользователя

- Использован компонент `Modal` из Ant Design
- Валидация: пустые поля, корректность ссылок
- Поле `id` недоступно при редактировании
- Кнопки блокируются при обращении к API
- После успешного создания/редактирования обновляется список пользователей

## Подготовка mockapi.io

1. Регистрация через GitHub или Google
2. Создание ресурса `users`
3. Настройка полей по умолчанию
4. Добавление тестовых пользователей

## Запуск проекта

1. git clone <ссылка на репозиторий>
2. cd <папка проекта>
3. npm install
4. Указать базовый url в env файле полученный от mockapi.io
5. npm run start
